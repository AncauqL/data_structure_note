# 数据结构复习课谨记知识点

## 1. 复杂度

- 时间复杂度：算法执行时间随问题规模 `n` 增长的阶。
- 大 O 表示法：`T(n)=O(f(n))`，只保留最高阶，忽略常数和低阶项。
- 顺序结构取最大：`O(f(n)) + O(g(n)) = max{O(f(n)), O(g(n))}`。
- 嵌套循环复杂度相乘，顺序循环复杂度相加。
- 常见时间复杂度顺序：
  `O(1) < O(log n) < O(n) < O(nlog n) < O(n^2) < O(n^3) < O(2^n)`。
- 空间复杂度：算法所需辅助存储空间随问题规模 `n` 增长的阶。

## 2. 线性表

- 顺序表：元素在内存中连续存放，支持随机访问。
- 顺序表插入：判断存储空间是否充足、插入位置是否合法，插入位置之后的元素整体后移。
- 顺序表删除：判断删除位置是否合法，删除位置之后的元素整体前移。
- 顺序表查找：注意起始位置和下标范围，不能越界。
- 单链表：每个结点由数据域和指针域组成。
- `p->data` 表示 `p` 所指结点的数据域，`p->link` 表示指针域。
- 动态分配结点：`malloc(sizeof(Lnode))`。
- 释放结点：`free(p)`。
- 头插法：每次把新结点插到头结点之后，生成顺序与输入顺序相反。
- 尾插法：每次把新结点插到链表尾部，需要尾指针跟踪尾结点。
- 单链表插入/删除的核心是修改指针，删除时还要释放结点内存。

## 3. 头结点、双向链表与循环链表

- 头结点：真实存在的结点，但一般不存有效数据。
- 首元结点：第一个真正存放数据的结点。
- 头指针：指向头结点的指针。
- 头结点作用：
  - 统一第一个元素和其他元素的插入操作。
  - 统一删除操作。
  - 允许链表为空。
- 双向链表结点结构：`prior | data | next`。
- 双向链表具有对称性：`p->prior->next = p = p->next->prior`。
- 双向链表删除结点 `p`：
  - `p->prior->next = p->next`
  - `p->next->prior = p->prior`
- 双向链表优点：找前驱和后继都方便。
- 循环链表：最后一个结点的指针指向头结点，形成环。
- 单链表常用结束条件：`p == NULL` 或 `p->link == NULL`。
- 循环链表常用结束条件：`p == L` 或 `p->link == L`。

## 4. 栈

- 栈：限定只能在表尾进行插入和删除的线性表。
- 栈顶：允许插入和删除的一端。
- 栈底：固定不变的一端。
- 特点：后进先出 `LIFO`，也称先进后出 `FILO`。
- 基本操作：置空栈、判空栈、取栈顶元素、入栈、出栈。
- 顺序栈入栈：判断栈是否满，元素进入栈顶，修改栈顶指针。
- 顺序栈出栈：判断栈是否空，从栈顶删除，修改栈顶指针。
- 链栈入栈：分配新结点，修改栈顶指针。
- 链栈出栈：判断是否为空，修改栈顶指针，释放被删结点。

## 5. 队列

- 队列：限定只能在一端插入、另一端删除的线性表。
- 队尾 `rear`：允许插入的一端。
- 队头 `front`：允许删除的一端。
- 特点：先进先出 `FIFO`。
- 链队列入队：新结点接到队尾，修改尾指针。
- 链队列出队：删除队头结点，修改头指针。
- 顺序队列中，队头和队尾都会移动。
- 顺序队列可能出现假溢出：数组前面有空位，但 `rear` 已到数组末尾。
- 循环队列通过取模运算实现循环：`(x + 1) % MAXSIZE`。
- 少用一个元素空间区分队空和队满：
  - 队空：`front == rear`
  - 队满：`(rear + 1) % MAXQSIZE == front`

## 6. 稀疏矩阵：十字链表

- 十字链表用于稀疏矩阵的链式压缩存储。
- 每个非零元结点包含五个域：
  - 行号 `row`
  - 列号 `col`
  - 值 `val`
  - 同行下一个非零元指针 `right`
  - 同列下一个非零元指针 `down`
- 十字链表包含行指针数组 `rhead` 和列指针数组 `chead`。
- 适合按行和按列都需要操作的稀疏矩阵问题。

## 7. 二叉树的存储结构

- 二叉树顺序存储：按完全二叉树的层次编号依次存放结点。
- 顺序存储特点：
  - 结点父子关系蕴含在存储位置中。
  - 适合满二叉树和完全二叉树。
  - 普通二叉树可能浪费空间。
- 二叉链表结点结构：`lchild | data | rchild`。
- 在 `n` 个结点的二叉链表中，有 `n + 1` 个空指针域。
- 二叉链表特点：找孩子容易，找双亲困难。
- 三叉链表在二叉链表基础上增加 `parent` 指针。
- 三叉链表特点：找孩子和找双亲都方便。
- 双亲数组存储：用数组记录每个结点的数据、双亲下标和左右孩子标志。

## 8. 二叉树遍历

- 二叉树遍历的三个基本任务：
  - 访问根结点。
  - 遍历左子树。
  - 遍历右子树。
- 先序遍历：根、左、右。
- 中序遍历：左、根、右。
- 后序遍历：左、右、根。
- 层序遍历：从上到下、从左到右，通常借助队列实现。
- 遍历题解题关键：
  - 先确定根结点。
  - 再划分左子树和右子树。
  - 对左右子树递归处理。

## 9. 线索二叉树

- 线索二叉树目的：利用二叉链表中的空指针域存储遍历前驱和后继。
- 在线索链表中，需要增加两个标志域 `LTag` 和 `RTag`。
- 若左子树不空：
  - `lchild` 指向左孩子。
  - `LTag = 0`，表示 `Link`。
- 若左子树为空：
  - `lchild` 指向遍历前驱。
  - `LTag = 1`，表示 `Thread`。
- 右指针同理：
  - `RTag = 0` 表示 `rchild` 指向右孩子。
  - `RTag = 1` 表示 `rchild` 指向遍历后继。

## 10. 哈夫曼树与哈夫曼编码

- 哈夫曼树构造步骤：
  1. 根据 `n` 个权值构造 `n` 棵只有根结点的二叉树。
  2. 每次选取两棵根结点权值最小的树作为左右子树。
  3. 新树根结点权值等于左右孩子权值之和。
  4. 删除原两棵树，将新树加入森林。
  5. 重复直到森林中只剩一棵树。
- 合并次数：`n - 1` 次。
- 有 `n` 个叶子结点的哈夫曼树共有 `2n - 1` 个结点。
- 带权路径长度：`WPL = Σ(权值 × 路径长度)`。
- 哈夫曼编码：
  - 左分支记为 `0`。
  - 右分支记为 `1`。
  - 每个字符的编码是根到叶子的 `0/1` 路径。
- 哈夫曼编码特点：
  - 任一字符编码都不是另一个字符编码的前缀。
  - 能保证正确解码。
  - 在给定频率下平均编码长度最短。

## 11. 图的存储结构与遍历

- 邻接表：为每个顶点建立一个单链表，再用顶点数组指向每个链表。
- 无向图中，顶点 `Vi` 的度等于第 `i` 个单链表中的结点数。
- 有向图中：
  - 顶点 `Vi` 的出度等于第 `i` 个单链表中的结点数。
  - 顶点 `Vi` 的入度等于所有单链表中邻接点域为 `i` 的结点数。
- 邻接表特点：找邻接点方便。
- 深度优先搜索 `DFS`：
  - 访问当前顶点。
  - 沿着一个未访问邻接点继续深入。
  - 通常用递归或栈实现。
- 广度优先搜索 `BFS`：
  - 访问当前顶点。
  - 依次访问所有未访问邻接点。
  - 通常用队列实现。

## 12. 最小生成树

- 最小生成树：连接图中所有顶点，且边权总和最小的生成树。
- Prim 算法思想：
  - 从一个顶点开始构造顶点集合。
  - 每次选取连接已选集合和未选集合的最小权边。
  - 将该边和对应顶点加入生成树。
- Kruskal 算法思想：
  - 初始时每个顶点自成一个连通分量。
  - 按边权从小到大选边。
  - 若边的两个顶点属于不同连通分量，则加入该边。
  - 若加入后形成回路，则舍弃该边。
  - 直到所有顶点连成一个连通分量。

## 13. 拓扑排序

- 拓扑排序用于解决具有优先关系的活动排序问题。
- 拓扑排序适用于有向无环图。
- 核心操作：
  - 求各顶点入度。
  - 输出入度为 `0` 的顶点。
  - 删除以该顶点为尾的弧。
  - 相关邻接点入度减 `1`。
- 若排序过程中仍有顶点未输出，但已没有入度为 `0` 的顶点，则图中存在环。

## 14. 最短路径

### Dijkstra 算法

- 用途：求单源最短路径。
- 初始时：
  - `S = {V0}`。
  - `T` 为其余顶点。
  - 若存在边 `<V0, Vi>`，距离为该边权值；否则为无穷大。
- 每次从 `T` 中选取当前距离最小的顶点 `W` 加入 `S`。
- 以 `W` 为中间顶点，尝试更新 `V0` 到其他顶点的最短距离。
- 重复直到所有顶点都加入 `S`。

### Floyd 算法

- 用途：求所有顶点对之间的最短路径。
- 图一般用邻接矩阵存储。
- `length[i][j]`：当前已知的 `i -> j` 最短路径长度。
- `path[i][j]`：最短路径 `i -> j` 上，顶点 `j` 的前一个顶点序号。
- 算法思想：逐个顶点作为中间顶点 `k` 进行试探。
- 若 `length[i][k] + length[k][j] < length[i][j]`，则更新：
  - `length[i][j] = length[i][k] + length[k][j]`
  - `path[i][j] = path[k][j]`
